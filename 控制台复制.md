问题分析
当从内嵌LoRA管理器删除最后一个LoRA时，消息无法同步到提示词。

根本原因：

在 prompt_index.vue 中处理 weilin_prompt_ui_update_lora_tags 消息时，只有当 loraTags.length > 0 时才处理：
if (loraTags && loraTags.length > 0) {
  // 更新提示词中的标签
}
当 loraTags 是空数组（删除最后一个LoRA时）时，这个条件不满足，所以不会清空提示词中的LoRA标签。

修复内容
添加对空数组的处理：
 else if (loraTags && loraTags.length === 0) {
  // 当 loraTags 为空数组时，清空提示词中的所有 LoRA 标签
  const currentText = inputText.value
  const wlrPattern = /<wlr:[^>]+>/g
  let cleanText = currentText.replace(wlrPattern, '')
  // 清理连续的逗号和空格
  cleanText = cleanText
    .replace(/,\s*,/g, ',')
    .replace(/,\s*$/g, '')
    .replace(/^\s*,/g, '')
    .trim()

  inputText.value = cleanText
  lastInputValue.value = inputText.value
  debouncedProcessInput()
}
