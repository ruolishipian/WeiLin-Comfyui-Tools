### 1. CSS布局冲突修复

**提交**: `2041975` - 修复CSS布局冲突：使用命名空间隔离防止插件样式污染

#### 问题描述
- `fastuse` 插件的 `.toolbar { height: 40px }` 样式导致本插件的 toolbar 高度被压缩
- `toolbar-bottom` 被挤出容器，被 `category-tabs` 覆盖
- 其他 ComfyUI 插件的 CSS 样式污染了本插件的 UI 布局

- 使用命名空间前缀：`.toolbar` → `.weilin-toolbar`
- 添加全局防御CSS：`[class^="weilin-"] { height: auto !important }`
- 强制覆盖外部样式：使用 `!important` 确保优先级

### 2. Lora堆按钮点击失效修复

**提交**: `af53056` - 修复WeiLinPromptUI节点"打开Lora堆"按钮无法点击的问题

#### 问题描述
1. **点击运行时提示词消失**：
   - 代码只更新了 `element.value`（DOM元素的值）
   - 但没有更新 `widget.value`（ComfyUI widget对象的值）
   - 当点击运行时，ComfyUI 序列化工作流，从 `widget.value` 获取值
   - 由于 `widget.value` 没有更新，所以获取到的是旧值

2. **按钮点击报错**：
   - `nodeTextAreaList[1]` 和 `nodeTextAreaList[3]` 可能为 undefined
   - 直接访问 `.value` 会导致 JavaScript 错误

#### 解决方案
- 新增 `nodeWidgetList` 数组：保存 widget 引用
- 同步更新 `widget.value`：每次更新 `element.value` 时，同时更新对应的 `widget.value`
- 添加安全检查：所有访问 `nodeTextAreaList` 的地方添加 undefined 检查

#### 修改文件
| 文件 | 修改内容 |
|------|---------|
| `js_node/weilin_prompt_ui_node.js` | 添加widget同步更新机制和安全检查 |

ComfyUI 2.0 dom-widget 容器遮挡问题总结
一、问题表现
按钮无法点击：节点的"打开提示词编辑器"和"打开Lora堆"按钮无法点击
画布遮挡：dom-widget容器遮挡了画布的大部分区域
影响节点放置：无法在遮挡区域放置其他节点
影响画布拖动：无法在遮挡区域拖动画布
二、问题根源
1. dom-widget 容器的异常状态
多个容器重叠：存在5个 dom-widget size-full 容器，几乎完全重叠
定位错误：所有容器都有 position: fixed，固定在视口上，而不是跟随节点
事件拦截：pointer-events: auto 导致容器拦截所有鼠标事件
尺寸异常：size-full 类导致容器覆盖整个画布
三、解决方案
 CSS 样式覆盖（js_node/weilin_fix.css）
 /* 针对包含weilin相关元素的dom-widget */
.dom-widget:has([class*="weilin-"]) {
    pointer-events: none !important;
    position: absolute !important;
}

/* 确保weilin内部的交互元素可以正常响应 */
.dom-widget:has([class*="weilin-"]) * {
    pointer-events: auto !important;
}

/* 针对包含特定name的textarea的dom-widget */
.dom-widget:has(textarea[name="positive"]) {
    pointer-events: none !important;
    position: absolute !important;
}

/* 确保textarea可以正常交互 */
.dom-widget:has(textarea[name="positive"]) textarea {
    pointer-events: auto !important;
}
 JavaScript 精确修复（只修复当前节点）
 const fixCurrentNodeDomWidgets = () => {
  if (!this.widgets) return;
  
  this.widgets.forEach(widget => {
    if (widget.element) {
      let parent = widget.element.parentElement;
      while (parent) {
        if (parent.classList && parent.classList.contains('dom-widget')) {
          // 设置pointer-events: none让画布可以交互
          parent.style.setProperty('pointer-events', 'none', 'important');
          // 设置position: absolute让容器跟随节点
          parent.style.setProperty('position', 'absolute', 'important');
          // 移除size-full类
          parent.classList.remove('size-full');
          // 确保内部元素可以交互
          if (widget.element) {
            widget.element.style.setProperty('pointer-events', 'auto', 'important');
          }
          break;
        }
        parent = parent.parentElement;
      }
    }
  });
};

def validate_conditioning_output(conditioning_data):
    """
    验证CONDITIONING输出格式是否正确。
    CONDITIONING应该是 [[cond_tensor, dict]] 或 None 的格式。
    """
    if conditioning_data is None:
        return True

    # 检查是否是列表格式
    if not isinstance(conditioning_data, list):
        print(f"[WARNING] CONDITIONING输出格式错误: 期望list，得到{type(conditioning_data)}")
        return False

    # 检查是否至少有一个元素
    if len(conditioning_data) == 0:
        print("[WARNING] CONDITIONING输出为空列表")
        return True

    # 检查第一个元素是否是 [cond, dict] 格式
    first_item = conditioning_data[0]
    if not isinstance(first_item, list) or len(first_item) < 2:
        print(f"[WARNING] CONDITIONING内部格式错误: 期望[cond, dict]，得到{type(first_item)}")
        return False

    return True
