# 性能优化修复方案

## 一、优化概述

本方案针对 `weilin-comfyui-tools` 项目中的性能问题,从以下六个维度进行系统性优化:

1. **减少深度watch使用** - 改用浅层watch或手动触发更新
2. **增加防抖时间** - 将processInput的防抖时间从50ms增加到100-200ms
3. **虚拟滚动** - 对大量标签列表使用虚拟滚动
4. **优化缓存策略** - 减少不必要的缓存清除
5. **异步处理** - 将翻译等耗时操作放入Web Worker
6. **批量更新** - 使用Vue的批量更新机制减少重渲染次数

## 二、详细优化方案

### 2.1 减少深度watch使用

#### 问题分析
当前代码中存在多处深度watch,会导致:
- 每次对象属性变化都触发重新计算
- 大型对象的深度遍历消耗大量性能
- 不必要的响应式更新

#### 优化方案

**文件位置**: `src/view/prompt_box/prompt_index.vue`

**当前代码** (第1280-1288行):
```javascript

  tokens,
  () => {
    if (!suppressUnsavedOnce) {
      unsavedChanges.value = true
    }
  },
  { deep: false } // 改为浅层watch,提升性能
)
```

**优化建议**:

### 2.4 优化缓存策略

#### 问题分析
当前缓存策略存在以下问题:
- 频繁清除缓存导致重复计算
- 缓存命中率低
- 不必要的缓存失效

#### 优化方案

**文件位置**: `src/view/prompt_box/prompt_index.vue`

**当前代码** (第1670-1681行):
```javascript
// 缓存计算样式,减少getComputedStyle调用

const getCachedStyles = (textarea) => {
  if (!cachedStyles) {
    cachedStyles = {
      width: window.getComputedStyle(textarea).width,
      font: window.getComputedStyle(textarea).font,
      padding: window.getComputedStyle(textarea).padding
    }
  }
  return cachedStyles
}
```

**优化后**:
```javascript
// 使用LRU缓存策略
class LRUCache {
  constructor(maxSize = 50) {
    this.maxSize = maxSize
    this.cache = new Map()
  }
  
  get(key) {
    if (this.cache.has(key)) {
      const value = this.cache.get(key)
      // 移到最后表示最近使用
      this.cache.delete(key)
      this.cache.set(key, value)
      return value
    }
    return null
  }
  
  set(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key)
    } else if (this.cache.size >= this.maxSize) {
      // 删除最久未使用的
      const firstKey = this.cache.keys().next().value
      this.cache.delete(firstKey)
    }
    this.cache.set(key, value)
  }
  
  clear() {
    this.cache.clear()
  }
}

// 创建缓存实例
const styleCache = new LRUCache(20)
const translationCache = new LRUCache(100)
const autocompleteCache = new LRUCache(50)

// 优化样式缓存
const getCachedStyles = (textarea) => {
  const cacheKey = `${textarea.className}-${textarea.offsetWidth}`
  let cached = styleCache.get(cacheKey)
  
  if (!cached) {
    cached = {
      width: window.getComputedStyle(textarea).width,
      font: window.getComputedStyle(textarea).font,
      padding: window.getComputedStyle(textarea).padding
    }
    styleCache.set(cacheKey, cached)
  }
  
  return cached
}

// 优化翻译缓存
const translateFunction = async (text, token) => {
  const cacheKey = `${text}-${currentLanguage.value}`
  let cached = translationCache.get(cacheKey)
  
  if (cached) {
    token.translate = cached
    return
  }
  
  // 执行翻译
  const result = await translatorApi.translate(text)
  translationCache.set(cacheKey, result)
  token.translate = result
}

// 优化自动补全缓存
const triggerAutocomplete = async (word) => {
  const cacheKey = word.toLowerCase()
  let cached = autocompleteCache.get(cacheKey)
  
  if (cached) {
    autocompleteResults.value = cached
    return
  }
  
  // 执行自动补全
  const results = await autocompleteApi.search(word)
  autocompleteCache.set(cacheKey, results)
  autocompleteResults.value = results
}
```

**缓存清除策略优化**:
```javascript
// 当前: 监听所有样式变化就清除缓存
onMounted(() => {
  const observer = new MutationObserver(() => {
    cachedStyles = null // 直接清除所有缓存
  })
  // ...
})

// 优化后: 只在相关样式变化时清除对应缓存
onMounted(() => {
  const observer = new MutationObserver((mutations) => {
    mutations.forEach((mutation) => {
      if (mutation.attributeName === 'style') {
        // 只清除样式相关缓存
        styleCache.clear()
      } else if (mutation.attributeName === 'class') {
        // 类名变化可能影响样式,清除样式缓存
        styleCache.clear()
      }
    })
  })
  
  if (inputAreaRef.value) {
    observer.observe(inputAreaRef.value, {
      attributes: true,
      attributeFilter: ['style', 'class']
    })
  }
  
  onBeforeUnmount(() => {
    observer.disconnect()
    // 组件卸载时清除所有缓存
    styleCache.clear()
    translationCache.clear()
    autocompleteCache.clear()
  })
})
```

### 2.5 异步处理 - Web Worker

#### 问题分析
翻译等耗时操作在主线程执行:
- 阻塞UI渲染
- 导致界面卡顿
- 用户体验差

#### 优化方案

**创建Web Worker文件**: `src/workers/translator.worker.js`

```javascript
// translator.worker.js
importScripts('https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js')

// 翻译API配置
const TRANSLATE_API = {
  alibaba: 'https://translate.alibaba.com/api/translate',
  bing: 'https://api.bing.microsofttranslator.com/v2/translate',
  youdao: 'https://openapi.youdao.com/api'
}

// 翻译缓存
const cache = new Map()

// 处理翻译请求
async function translate(text, from, to, service) {
  const cacheKey = `${text}-${from}-${to}-${service}`
  
  // 检查缓存
  if (cache.has(cacheKey)) {
    return cache.get(cacheKey)
  }
  
  try {
    // 根据服务选择API
    const apiUrl = TRANSLATE_API[service]
    const response = await fetch(`${apiUrl}?text=${encodeURIComponent(text)}&from=${from}&to=${to}`)
    const result = await response.json()
    
    // 缓存结果
    cache.set(cacheKey, result)
    
    // 限制缓存大小
    if (cache.size > 200) {
      const firstKey = cache.keys().next().value
      cache.delete(firstKey)
    }
    
    return result
  } catch (error) {
    console.error('Translation error:', error)
    return null
  }
}

// 监听主线程消息
self.onmessage = async function(e) {
  const { type, data, id } = e.data
  
  switch (type) {
    case 'translate':
      const result = await translate(data.text, data.from, data.to, data.service)
      self.postMessage({ type: 'translate-result', data: result, id })
      break
      
    case 'batch-translate':
      const results = await Promise.all(
        data.items.map(item => translate(item.text, data.from, data.to, data.service))
      )
      self.postMessage({ type: 'batch-translate-result', data: results, id })
      break
      
    case 'clear-cache':
      cache.clear()
      self.postMessage({ type: 'cache-cleared', id })
      break
  }
}
```

**在Vue组件中使用Web Worker**: `src/view/prompt_box/prompt_index.vue`

```javascript
// 创建Worker实例
let translatorWorker = null

onMounted(() => {
  // 初始化Worker
  translatorWorker = new Worker(
    new URL('@/workers/translator.worker.js', import.meta.url),
    { type: 'module' }
  )
  
  // 监听Worker消息
  translatorWorker.onmessage = (e) => {
    const { type, data, id } = e.data
    
    switch (type) {
      case 'translate-result':
        // 处理翻译结果
        handleTranslateResult(data, id)
        break
        
      case 'batch-translate-result':
        // 处理批量翻译结果
        handleBatchTranslateResult(data, id)
        break
    }
  }
})

onBeforeUnmount(() => {
  // 清理Worker
  if (translatorWorker) {
    translatorWorker.terminate()
    translatorWorker = null
  }
})

// 使用Worker进行翻译
const translateFunction = (text, token) => {
  const requestId = Date.now()
  
  // 发送翻译请求到Worker
  translatorWorker.postMessage({
    type: 'translate',
    data: {
      text,
      from: 'auto',
      to: currentLanguage.value,
      service: currentTranslateService.value
    },
    id: requestId
  })
  
  // 存储请求ID,用于匹配结果
  pendingTranslations.set(requestId, { token, text })
}

// 处理翻译结果
const handleTranslateResult = (result, id) => {
  const pending = pendingTranslations.get(id)
  if (pending && result) {
    pending.token.translate = result.translation
    pendingTranslations.delete(id)
  }
}

// 批量翻译
const batchTranslate = (texts) => {
  const requestId = Date.now()
  
  translatorWorker.postMessage({
    type: 'batch-translate',
    data: {
      items: texts.map(text => ({ text })),
      from: 'auto',
      to: currentLanguage.value,
      service: currentTranslateService.value
    },
    id: requestId
  })
}
```

### 2.6 批量更新

#### 问题分析
当前代码中存在多次独立更新:
- 每次更新都触发重新渲染
- 没有利用Vue的批量更新机制
- 性能浪费

#### 优化方案

**文件位置**: `src/view/prompt_box/prompt_index.vue`

**优化策略1: 使用nextTick批量更新**

```javascript
// 当前: 多次独立更新
const updateMultipleTokens = (updates) => {
  updates.forEach(({ index, text }) => {
    tokens.value[index].text = text
    updateInputText() // 每次都更新
  })
}

// 优化后: 批量更新
const updateMultipleTokens = async (updates) => {
  // 批量修改数据
  updates.forEach(({ index, text }) => {
    tokens.value[index].text = text
  })
  
  // 等待下一个tick,一次性更新
  await nextTick()
  updateInputText()
}
```

## 四、实施计划

### 4.1 优先级排序

| 优先级 | 优化项 | 预期收益 | 实施难度 | 风险 |
|--------|--------|----------|----------|------|
| P0 | 增加防抖时间 | 高 | 低 | 低 |
