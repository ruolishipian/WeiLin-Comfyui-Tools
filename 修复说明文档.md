### 1. CSS布局冲突修复

**提交**: `2041975` - 修复CSS布局冲突：使用命名空间隔离防止插件样式污染

#### 问题描述
- `fastuse` 插件的 `.toolbar { height: 40px }` 样式导致本插件的 toolbar 高度被压缩
- `toolbar-bottom` 被挤出容器，被 `category-tabs` 覆盖
- 其他 ComfyUI 插件的 CSS 样式污染了本插件的 UI 布局

#### 解决方案
- 使用命名空间前缀：`.toolbar` → `.weilin-toolbar`
- 添加全局防御CSS：`[class^="weilin-"] { height: auto !important }`
- 强制覆盖外部样式：使用 `!important` 确保优先级

#### 修改文件
| 文件 | 修改内容 |
|------|---------|
| `src/src/view/tag_manager/tag_index.vue` | HTML模板和CSS样式添加命名空间前缀 |
| `src/src/view/cloud/index.vue` | HTML模板和CSS样式添加命名空间前缀 |
| `dist/style.css` | 编译输出更新 |
| `dist/javascript/main.entry.js` | 编译输出更新 |

#### 代码示例
```css
/* 全局防御：防止其他插件覆盖 weilin 命名空间的样式 */
[class^="weilin-"],
[class*=" weilin-"] {
  height: auto !important;
  box-sizing: border-box !important;
}

/* 防御性样式：确保不被其他插件覆盖 */
.weilin-toolbar {
  height: auto !important;
  min-height: 40px !important;
  flex-direction: column;
  padding: 12px 16px;
  box-sizing: border-box !important;
}
```

### 2. Lora堆按钮点击失效修复

**提交**: `af53056` - 修复WeiLinPromptUI节点"打开Lora堆"按钮无法点击的问题

#### 问题描述
1. **点击运行时提示词消失**：
   - 代码只更新了 `element.value`（DOM元素的值）
   - 但没有更新 `widget.value`（ComfyUI widget对象的值）
   - 当点击运行时，ComfyUI 序列化工作流，从 `widget.value` 获取值
   - 由于 `widget.value` 没有更新，所以获取到的是旧值

2. **按钮点击报错**：
   - `nodeTextAreaList[1]` 和 `nodeTextAreaList[3]` 可能为 undefined
   - 直接访问 `.value` 会导致 JavaScript 错误

#### 解决方案
- 新增 `nodeWidgetList` 数组：保存 widget 引用
- 同步更新 `widget.value`：每次更新 `element.value` 时，同时更新对应的 `widget.value`
- 添加安全检查：所有访问 `nodeTextAreaList` 的地方添加 undefined 检查

#### 修改文件
| 文件 | 修改内容 |
|------|---------|
| `js_node/weilin_prompt_ui_node.js` | 添加widget同步更新机制和安全检查 |

**提交**: `1b393b0` - fix: 修复Lora堆按钮点击失效及缓存问题

#### 问题描述
- `openLoraManager` 函数参数处理不当，可能接收到事件对象而非DOM元素
- 内联 `onclick` 事件被 LiteGraph 拦截

#### 解决方案

## 技术要点总结

### 1. CSS命名空间隔离
- 使用 `weilin-` 前缀命名所有CSS类
- 添加全局防御规则 `[class^="weilin-"]`
- 使用 `!important` 确保样式优先级

### 2. ComfyUI Widget同步机制
- 同时更新 `element.value` 和 `widget.value`
- 使用 `Object.defineProperty` 监听属性变化
- 通过 `postMessage` 实现跨窗口通信

### 3. 事件处理优化
- 使用 `addEventListener` 替代内联 `onclick`
- 添加事件冒泡阻止，防止被框架拦截
- 正确处理事件对象和DOM元素

### 4. 数据结构优化
- 为目录结构添加 `all` 键存储完整文件列表
- 兼容新旧数据结构，确保向后兼容

### 5. 缓存控制
- 静态资源URL添加版本号参数
- 修改版本号强制浏览器重新加载
