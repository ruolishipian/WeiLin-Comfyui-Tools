ComfyUI 2.0 中的按钮创建主要分为前端（Web 界面） 和后端（Python 逻辑） 两个层面，核心是前端通过 React + TypeScript 构建交互按钮，后端通过 FastAPI 提供按钮触发的接口支持。以下是具体实现方式：
一、前端按钮创建（核心）
ComfyUI 2.0 前端基于 React 18 + TypeScript 重构，按钮主要分为两类：通用功能按钮（如运行、保存）和节点内按钮（自定义节点交互），核心实现逻辑如下：
1. 通用功能按钮（全局 / 面板按钮）
这类按钮定义在 frontend/src/components/ 目录下（如 Toolbar.tsx、QueueButtons.tsx），使用 React 组件 + 样式模块实现：
tsx
// frontend/src/components/QueueButtons.tsx（简化示例）
import React from 'react';
import { useApi } from '../hooks/useApi'; // 封装 API 调用
import styles from './QueueButtons.module.css'; // CSS 模块化

export const QueueButtons = () => {
  const { api } = useApi(); // 获取 API 客户端

  // 按钮点击事件（调用后端接口）
  const handleRunPrompt = async () => {
    await api.post('/prompt', { prompt: window.appState.currentPrompt });
  };

  const handleClearQueue = async () => {
    await api.delete('/queue');
  };

  // 渲染按钮
  return (
    <div className={styles.queueButtons}>
      {/* 运行按钮 - 核心功能按钮 */}
      <button 
        className={`${styles.btn} ${styles.btnPrimary}`}
        onClick={handleRunPrompt}
        disabled={!window.appState.hasPrompt}
      >
        运行 Prompt
      </button>

      {/* 清空队列按钮 */}
      <button 
        className={`${styles.btn} ${styles.btnSecondary}`}
        onClick={handleClearQueue}
      >
        清空队列
      </button>
    </div>
  );
};
关键说明：
样式：使用 CSS Modules（.module.css）隔离样式，避免全局污染；
交互：点击事件通过封装的 useApi Hook 调用后端 FastAPI 接口；
状态：通过 window.appState 或 React 状态管理按钮禁用 / 激活状态。
2. 节点内自定义按钮（扩展节点）
自定义节点中的按钮是开发者最常接触的场景，ComfyUI 2.0 提供了标准化的节点按钮创建方式，定义在节点的 defineNode 配置中：
tsx
// 自定义节点示例（frontend/src/custom_nodes/MyCustomNode.tsx）
import { defineNode } from 'comfyui/node';

export const MyCustomNode = defineNode({
  name: 'MyCustomButtonNode',
  displayName: '自定义按钮节点',
  inputs: [],
  outputs: [],
  // 节点UI渲染函数
  ui: (node, context) => {
    return (
      <div className="node-controls">
        {/* 节点内按钮 */}
        <button 
          className="comfy-btn comfy-btn-sm" // 内置样式类
          onClick={() => {
            // 按钮点击逻辑：可调用后端接口/修改节点状态
            fetch('/custom-node-action', {
              method: 'POST',
              body: JSON.stringify({ nodeId: node.id }),
            });
            // 或修改节点本地状态
            node.data.buttonClicked = true;
            context.updateNode(node); // 更新节点渲染
          }}
        >
          节点自定义按钮
        </button>
      </div>
    );
  },
});
内置样式类（ComfyUI 2.0 标准化）：
comfy-btn：基础按钮样式
comfy-btn-sm：小尺寸按钮
comfy-btn-primary：主色调按钮
comfy-btn-danger：危险操作按钮（如删除）
二、后端按钮接口支持（触发逻辑）
前端按钮的点击事件最终需要后端接口承接，ComfyUI 2.0 通过 FastAPI 定义接口：
python
运行
# comfy/server/routes/custom_routes.py
from fastapi import APIRouter
from comfy.server.server import app

# 方式1：创建独立路由
router = APIRouter(prefix="/custom-node-action")

@router.post("")
async def handle_custom_button_click(nodeId: str):
    # 处理按钮点击逻辑（如执行节点操作、修改状态）
    return {"status": "success", "nodeId": nodeId}

# 注册到 app 对象
app.include_router(router)

# 方式2：直接挂载到 app（简化）

三、核心补充
按钮状态同步：ComfyUI 2.0 引入 Socket.IO 实现前后端实时同步（如按钮点击后队列状态更新）；
移动端适配：按钮样式通过 rem 单位 + 媒体查询适配不同屏幕；
扩展按钮最佳实践：
优先使用内置样式类，保持界面一致性；
按钮点击逻辑异步化，避免阻塞前端界面；
关键操作添加确认弹窗（如删除按钮）。
ComfyUI Nodes 2.0 测试版的核心区别，在于前端架构从 LiteGraph.js 迁移至 Vue.js 驱动的新节点系统，开启后会启用新交互与性能能力，但伴随兼容性与稳定性风险；关闭则回退旧版，稳定但无新特性ComfyUI。
核心差异速览
表格
维度	开启 Nodes 2.0	关闭 Nodes 2.0（旧版）
底层架构	Vue.js 驱动，支持动态组件 / 可扩展节点	LiteGraph.js + Canvas2D 传统架构ComfyUI
功能与性能	支持动态控件、自适应节点尺寸、更快功能迭代ComfyUI	功能稳定，无新交互能力，迭代慢ComfyUI
兼容性	部分旧自定义节点 / 扩展可能不可用	兼容绝大多数旧节点 / 扩展，生态稳定ComfyUI
稳定性	测试中，存在已知 UI / 交互瑕疵（如缩放模糊、弹窗层级）	成熟稳定，无已知重大问题ComfyUI
切换方式	点击 ComfyUI 图标 → 菜单中开启 / 关闭ComfyUI	同上，即时切换，无需重启ComfyUI
一、开启 Nodes 2.0 的收益与代价
✅ 核心收益
新交互与能力
可扩展节点、动态控件（如自适应尺寸、更灵活的参数输入）ComfyUI
更现代的组件与交互模式（如悬浮菜单、状态同步更流畅）ComfyUI
为后续「线性模式」等下一代工作流打基础ComfyUI
开发效率提升
前端迭代更快，新功能 / 修复可快速上线
自定义节点开发更友好，降低 UI 拓展门槛
⚠️ 潜在代价
兼容性风险
部分旧版自定义节点、第三方扩展（如 Manager、Model Finder 等）可能无法加载或异常
未适配 2.0 的扩展可能出现节点错位、样式丢失、功能失效
稳定性与体验瑕疵（测试期）
已知问题：缩放时文字短暂模糊、下拉菜单层级异常、数值调整交互不直观、部分输入框不可见等
部分功能（如 Stop/Cancel、Clear Queue）初期缺失，已逐步修复ComfyUI
二、关闭 Nodes 2.0 的适用场景
追求稳定：当前工作流依赖大量旧扩展 / 自定义节点，无法承受兼容性风险ComfyUI
避开测试期瑕疵：对 UI 交互、视觉一致性要求高，暂不接受已知问题
紧急任务：需快速出图，不希望因版本切换影响进度
三、切换与使用建议
切换方式
点击界面左上角 ComfyUI 图标 → 打开设置菜单 → 找到「Nodes 2.0」开关，即时生效，无需重启ComfyUI
使用建议
日常稳定使用：建议关闭，确保生态兼容与流程稳定ComfyUI
测试新功能 / 开发自定义节点：可开启，配合日志排查异常，完成后切回旧版ComfyUI
开启后遇异常：优先禁用可疑扩展 / 自定义节点，必要时切回旧版临时规避
